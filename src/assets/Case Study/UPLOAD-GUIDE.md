## How to Upload These to Your Site

### If you use WordPress
1. Create a new page for each case study
2. Set the page title to the H1 heading in the file
3. Paste the content — your AI assistant or page builder can handle the markdown formatting
4. Set the URL slug to match the filename (e.g. `/case-study-auto-repair-seo`)
5. Add an internal link from your homepage or services page to each case study

### If you use a page builder (Webflow, Framer, Wix, etc.)
1. Create a "Case Studies" section or page
2. Feed each file to your AI tool and ask it to format it for your page builder
3. Keep headings, bold text, and table structure intact

### If you use a custom site
1. These are clean markdown files — any markdown parser will render them correctly
2. Alternatively feed them to your AI as plain text with the instruction: "format this as a webpage using our site's design"

## SEO Instructions for Each Page

When uploading, set these fields for each page:

**Page title (browser tab):**
Use the H1 heading + " | Surg Solutions"
Example: "How We Took a Local Repair Shop from 8 Google Visitors to 624 | Surg Solutions"

**Meta description (160 characters max):**
Use the first sentence under "The Problem" section, trimmed to fit.

**URL slug:**
Use the filename without the .md extension.
Example: `/case-study-auto-repair-seo`

**Internal linking:**
- Link to each case study from your Services page
- Link to your Contact or Pricing page from each case study's CTA section
- Link between related case studies where relevant
